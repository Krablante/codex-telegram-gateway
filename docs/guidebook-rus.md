# Spike: guidebook для новичка

Это короткий практический гид по боту. Не полный справочник по всем флагам, а карта старта.

## Быстрый старт

`General` — это лобби. Обычно там ты делаешь три вещи:

- открываешь глобальное меню через `/global`
- создаёшь новые рабочие топики через `/new [host=<id>] [cwd=...|path=...] <title>`; без `host=` используется configured current/default host, обычно `CURRENT_HOST_ID`, а global `New Topic` пропускает выбор хоста только когда в registry ровно один host
- используешь общие операторские команды вроде `/help`, `/guide`, `/hosts` или `/zoo`

Обычный старт выглядит так:

```text
/new Backend Cleanup
/new host=worker-a cwd=homelab/infra/automation/codex-telegram-gateway Gateway Audit
```

Потом заходишь в новый топик и отправляешь задачу обычным текстом.

## Кто что делает

`Spike` — это рабочий бот. Он читает код, правит файлы, запускает команды и отвечает по задаче.

Второго supervisory-бота больше нет. Нормальный путь теперь — прямой topic-local workflow с одним worker и одной сессией на топик.

## Почему меню должны быть дефолтом

Самый удобный путь такой:

- `/global` в `General` для общих настроек
- `/menu` внутри рабочего топика для локальных настроек
- pinned menu в `Zoo`

Меню — более безопасный и быстрый путь для стабильных настроек:

- посмотреть `Status`
- сменить язык интерфейса
- включить или выключить `wait`
- поменять модель/ризонинг Spike
- поменять отдельный профиль summarizer для `/compact`
- запустить `Compact`, `Interrupt` и `Purge`
- настроить suffix

Практическое правило: если хочешь менять стабильную настройку — сначала иди в меню.

## Что меню не заменяют

Меню не заменяют саму работу. Эти вещи всё ещё лучше делать прямым текстом или командами:

- обычный prompt в рабочем топике
- создание темы через `/new ...`
- `/q`, когда ты явно хочешь “это сделать следующим”
- `/diff`, чтобы посмотреть git-изменения
- `/guide`, `/help`, `/zoo` и `/clear`, если тебе удобнее прямые команды

Некоторые значения всё ещё стартуют из меню, но вводятся следующим обычным текстовым сообщением — например custom wait или свободный текст suffix.

## Как обычно идёт работа в топике

Внутри рабочего топика обычный текст считается нормальным prompt. Если run свободен — стартует сразу. На дефолтном `exec-json` follow-up во время активного run принимается как live steer в тот же логический run. Используй `/q`, когда явно хочешь выполнить текст после завершения текущего turn.

`/q` — это не “ещё одна мысль”, а явное “сделай это следующим”.

```text
/q After the current check finishes, prepare a README patch.
```

Если ничего не идёт, такой prompt обычно стартует сразу. Если текущий run ещё закрывается, `/q` спокойно ждёт своей очереди.

## Когда нужен /wait

`/wait` нужен, когда ты хочешь собрать один большой prompt из нескольких сообщений.

```text
/wait 60
```

После этого отправляешь несколько сообщений и сбрасываешь их отдельным сообщением:

```text
All
Все
Всё
```

Для повседневной работы обычно хватает локального `/wait`. Если такое же окно нужно для всего чата, есть `/wait global`.

## Зачем нужен /suffix

`/suffix` нужен для стабильных привычек ответа, например:

- отвечай кратко
- говори, что именно проверил
- не переводи команды и код
- упоминай, запускались ли тесты

Если правило относится только к одному топику — используй topic suffix. Если это твоя привычка для всего чата — global suffix. Для разовой инструкции обычный prompt обычно проще.

## Zoo без лишней сложности

`/zoo` — это не обычный рабочий топик. Это отдельная операторская доска по проектам.

Обычный flow почти полностью menu-only:

1. Открываешь `/zoo`.
2. Нажимаешь `Add project`.
3. Описываешь проект.
4. Подтверждаешь предложенный root.

После этого топик в основном живёт как один pinned menu с pet cards и кнопками `Back`, `Refresh`, `Remove` и `Respawn menu`.

## Что реально стоит помнить

Если не хочется держать в голове много команд, вот минимальный набор:

- `/new` — создать новый рабочий топик
- `/global` — открыть глобальное меню в `General`
- `/menu` — открыть локальное меню топика
- `/status` — быстрый чек темы
- `/q` — поставить следующий prompt в очередь
- `/diff` — посмотреть, что уже изменилось в git workspace
- `/compact` — сжать длинную тему в brief
- `/guide` — получить этот guidebook в PDF
- `/zoo` — открыть доску проектов
- `/interrupt` — остановить run, когда это действительно нужно

## Хорошие привычки

- один топик = один поток работы
- не используй `General` как обычную рабочую тему
- стабильные настройки меняй через меню
- саму задачу пиши обычными сообщениями
- `/q` используй только для явного следующего шага
- не затягивай с `/compact`, когда тема становится большой

Этого уже достаточно, чтобы уверенно пользоваться ботом без погружения во внутренности.
